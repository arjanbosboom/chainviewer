# Contributing

## Scope

This project is in early setup. Keep changes focused, reviewable, and aligned with the multi-chain architecture described in the project plan.

## Guidelines

1. Open an issue for larger changes before implementation.
2. Keep pull requests small and single-purpose.
3. Prefer maintainable designs over clever abstractions.
4. Keep chain-specific behavior isolated behind dedicated adapters.
5. Update documentation when behavior or structure changes.

## Development

1. Use Docker Compose for local development when possible.
2. Follow the repository structure already in place.
3. Avoid introducing coin-specific assumptions into shared layers.

## Pull Requests

1. Describe the problem and the change clearly.
2. Include verification steps.
3. Note any follow-up work that remains.
